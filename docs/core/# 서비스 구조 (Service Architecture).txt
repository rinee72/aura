# 서비스 구조 (Service Architecture)

## 1. 계정 구조
### ▷ 팬(Fan)
- 질문 생성
- 팬 커뮤니티 이용
- 셀럽 구독

### ▷ 셀럽(Celebrity)
- 질문 큐레이션 리스트 확인
- 답변 작성
- 피드 작성

### ▷ 매니저(Manager)
- 셀럽 콘텐츠 관리
- 팬 질문 모니터링 및 필터링
- 이슈 질문 숨기기

---

## 2. 서비스 흐름 구조

### 2.1 팬 → 질문 카드 → 큐레이션
1. 팬이 질문 작성  
2. 욕설 필터 1차 체크  
3. 질문이 전체 팬에게 공개  
4. 팬들이 좋아요 기반으로 질문 우선순위 형성  
5. 상위 몇 개가 셀럽에게 전달되는 '질문 큐레이션 리스트'로 노출

---

### 2.2 셀럽 → 정제된 질문 확인 → 답변
1. 셀럽은 오늘의 상위 질문 리스트만 확인  
2. 질문 선택 후 답변 작성  
3. 답변은 구독 팬 전체에게 공개  

---

### 2.3 매니저 → 관리 기능
1. 팬 질문 실시간 모니터링  
2. 민감/유해 질문을 셀럽 보기 전에 숨기기  
3. 이슈 발생 시 즉시 차단  
4. 셀럽 프로필/콘텐츠 관리

---

## 3. 화면 구조 (Information Architecture)

### 3.1 팬
- 홈(셀럽 최신 답변)
- 질문 보드
- 팬 커뮤니티
- 마이페이지
- 셀럽 리스트(구독)

### 3.2 셀럽
- 오늘의 질문 Top 10
- 내 답변 피드
- 콘텐츠 작성
- 계정 설정

### 3.3 매니저
- 질문 모니터링 대시보드
- 문제성 질문 숨기기 목록
- 셀럽 설정
- 신고/필터링 리포트

---

## 4. 기술 구조(High-level)
- Frontend: React / Next.js  
- Backend: Node.js or Nest.js  
- DB: PostgreSQL  
- Realtime: WebSocket or Server-Sent Events  
- Filter Module: 사전 기반 욕설 필터 + 정규식  
- Role-based Access Control(RBAC) 기반 3계정 구조  
- 이미지/콘텐츠 저장: Cloud storage (S3 등)

---

## 5. 데이터 흐름 요약
1. 팬 → 질문 제출  
2. 서버 → 욕설 필터링  
3. DB 저장 → 팬 질문 보드 반영  
4. 팬들의 좋아요 → 실시간 질문 랭킹  
5. 셀럽 → Top 10 질문만 확인  
6. 셀럽 답변 → 팬 전체에게 공개

요약: **팬의 활동은 공개 공간에서 정제되고, 셀럽은 ‘정제된 결과물’만 본다.**
