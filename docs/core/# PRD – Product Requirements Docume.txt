# PRD – Product Requirements Document

## 1. 제품 목표
셀럽과 팬이 서로 건강하게 소통할 수 있는 환경을 구축하고  
셀럽의 정신적 스트레스, 악성 메시지로부터의 리스크를 최소화하는 것을 목표로 한다.

---

## 2. 사용자 유형 및 페르소나

### 2.1 셀럽
- 피로도 낮은 소통을 원함
- 악플/스팸 최소화
- 팬 질문만 선별적으로 확인하고 싶음

### 2.2 팬
- 셀럽의 진솔한 답변을 보고 싶음
- 중복되지 않는 효율적 질문 시스템 필요
- 자유롭게 활동할 팬 커뮤니티 공간 필요

### 2.3 매니저/소속사
- 셀럽의 콘텐츠 및 팬 메시지 모니터링
- 이슈성 질문/메시지 조기 차단
- 셀럽 보호 기능 관리

---

## 3. 주요 기능 요구사항

### 3.1 팬 기능
- 회원가입/로그인
- 셀럽 구독
- 질문 카드 작성
- 다른 팬 질문 좋아요
- 팬 커뮤니티 참여(글/댓글)

### 3.2 셀럽 기능
- 오늘의 인기 질문(큐레이션 리스트) 확인
- 질문 선택 후 답변 게시
- 셀럽만의 피드 작성
- 악성 메시지 숨김(기본 필터링)

### 3.3 매니저 기능
- 팬 질문 모니터링
- 민감/악성 질문 숨기기
- 셀럽 계정 관리
- 필터링된 메시지 리포트 확인

---

## 4. 비기능 요구사항

### 4.1 성능
- 질문 리스트는 1초 이하로 로딩
- 셀럽 답변은 실시간 반영 가능(웹소켓 또는 폴링)

### 4.2 안정성
- 악성 메시지 기본 필터링은 90% 이상 탐지율
- 3계정 권한 구조 구분 명확히 적용

### 4.3 보안
- JWT 기반 인증
- 팬→셀럽 메시지는 서버에서 필터링 후 전달

### 4.4 확장성
- 향후 AI 모델, 멘탈 모듈 추가가 가능하게 구조 설계

---

## 5. 성공 지표 (KPI)
- 팬 질문 중복률 70% 감소
- 셀럽 피로도(확인해야 할 메시지 수) 90% 감소
- 악성 메시지 노출 80% 감소
- 팬 재방문율 50% 이상

---

## 6. MVP 범위 (최소 기능)
- 팬 질문 카드 작성/좋아요
- 질문 정렬(좋아요순)
- 셀럽 질문 선택 및 답변 기능
- 기본 욕설/비속어 필터
- 유저 3계정 구조(팬/셀럽/매니저)
- 커뮤니티 게시판(팬 전용)
