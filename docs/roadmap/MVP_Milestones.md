# AURA MVP 개발 마일스톤 전략

## 📋 프로젝트 개요
**목표**: 셀럽-팬 간 건강한 소통 플랫폼 MVP 구축  
**기술 스택**: Flutter + Supabase  
**핵심 가치**: 셀럽 피로도 최소화 + 팬 경험 극대화 + 안전한 소통 환경

---

## 🎯 Milestone 구조 Overview

```
M0 (Foundation) → M1 (Core Infrastructure) → M2 (Fan Features) 
→ M3 (Celebrity Features) → M4 (Manager Features) → M5 (Launch)
```

**총 개발 기간**: 약 12-16주 (3-4개월)

---

## 🏗️ Milestone 0: 프로젝트 기반 구축
**기간**: 1주  
**목표**: 개발 환경 및 기본 아키텍처 설정

### 주요 산출물
- Flutter 프로젝트 초기화 (멀티 플랫폼 설정)
- Supabase 프로젝트 생성 및 환경 구성
- Git 저장소 및 브랜치 전략 수립
- CI/CD 파이프라인 기본 설정
- 개발/스테이징/프로덕션 환경 분리
- 디자인 시스템 기본 컴포넌트 라이브러리
- 프로젝트 폴더 구조 및 코딩 컨벤션 문서화

### 완료 조건
- [ ] Flutter 앱이 iOS/Android/Web에서 실행 가능
- [ ] Supabase 연결 테스트 성공
- [ ] 팀원 모두 개발 환경 세팅 완료

### 의존성
- 없음 (프로젝트 시작점)

---

## 🔐 Milestone 1: 인증 및 핵심 인프라
**기간**: 2주  
**목표**: 3-Tier 계정 구조 및 데이터베이스 설계 완성

### 주요 산출물
#### 1.1 데이터베이스 스키마
- Users 테이블 (role: fan/celebrity/manager)
- Questions 테이블
- Answers 테이블
- Communities 테이블
- Subscriptions 테이블
- RLS(Row Level Security) 정책 설정

#### 1.2 인증 시스템
- Supabase Auth 연동
- 회원가입/로그인 화면
- JWT 기반 세션 관리
- Role-based Access Control (RBAC) 구현
- 소셜 로그인 (선택: Google/Apple)

#### 1.3 기본 Navigation
- 역할별 화면 분기 라우팅
- 공통 UI 컴포넌트 (AppBar, BottomNav 등)

### 완료 조건
- [ ] 팬/셀럽/매니저 각각 회원가입 후 역할에 맞는 화면 진입
- [ ] DB 스키마가 ERD 문서와 일치
- [ ] RLS 정책으로 권한 분리 검증 완료

### 의존성
- M0 완료 필수

---

## 👥 Milestone 2: 팬 핵심 기능 구현
**기간**: 3주  
**목표**: 팬이 질문하고, 답변을 보고, 커뮤니티를 이용할 수 있는 환경 구축

### 주요 산출물
#### 2.1 질문 카드 시스템
- 질문 작성 화면 (텍스트 입력)
- 욕설/비속어 1차 필터링 (클라이언트)
- 질문 목록 화면 (전체 공개)
- 질문 상세 화면

#### 2.2 좋아요 및 정렬
- 질문 좋아요 기능 (실시간 반영)
- 좋아요순 정렬 알고리즘
- 무한 스크롤 or 페이지네이션

#### 2.3 셀럽 구독 시스템
- 셀럽 리스트 화면
- 셀럽 프로필 화면
- 구독/구독취소 기능
- 내 구독 목록 관리

#### 2.4 답변 피드
- 셀럽 답변 목록 (홈 피드)
- Q&A 형태로 질문-답변 연결 표시
- 실시간 업데이트 (Supabase Realtime)

#### 2.5 팬 커뮤니티
- 게시글 목록/작성/수정/삭제
- 댓글 기능
- 검색 및 필터링

### 완료 조건
- [ ] 팬이 질문 작성하고 다른 팬 질문에 좋아요 가능
- [ ] 욕설 포함 질문은 제출 차단
- [ ] 좋아요순 정렬이 실시간으로 작동
- [ ] 셀럽 구독 후 해당 셀럽 답변만 필터링 확인 가능
- [ ] 커뮤니티 글 작성/댓글 정상 작동

### 의존성
- M1 (인증 및 DB) 완료 필수

---

## 🎤 Milestone 3: 셀럽 핵심 기능 구현
**기간**: 2.5주  
**목표**: 셀럽이 정제된 질문을 확인하고 답변할 수 있는 시스템 구축

### 주요 산출물
#### 3.1 질문 큐레이션 대시보드
- 오늘의 Top 질문 리스트 (좋아요 기준)
- 날짜별 필터링
- 질문 상태 관리 (미답변/답변완료)

#### 3.2 답변 작성 시스템
- 질문 선택 → 답변 작성 화면
- 텍스트 답변 입력
- 답변 미리보기
- 답변 게시/임시저장

#### 3.3 답변 관리
- 내 답변 목록
- 답변 수정/삭제
- 답변 통계 (조회수, 좋아요 등)

#### 3.4 셀럽 피드 작성
- 일반 피드 작성 기능
- 이미지 업로드 (Supabase Storage)
- 피드 수정/삭제

#### 3.5 셀럽 프로필 관리
- 프로필 이미지/소개 수정
- 구독자 수 표시
- 계정 설정

### 완료 조건
- [ ] 셀럽이 좋아요 Top 10 질문 확인 가능
- [ ] 질문 선택 후 답변 작성 및 게시 성공
- [ ] 답변이 팬 피드에 실시간 반영
- [ ] 셀럽 피드 작성 및 이미지 업로드 정상 작동
- [ ] 답변 안 한 질문은 계속 노출, 답변한 질문은 '답변완료' 상태 변경

### 의존성
- M2 (팬 기능) 완료 필수 (질문 데이터가 있어야 큐레이션 가능)

---

## 🛡️ Milestone 4: 매니저 기능 및 보안 강화
**기간**: 2주  
**목표**: 매니저의 모니터링/관리 기능 및 AI 악플 필터링 구현

### 주요 산출물
#### 4.1 매니저 대시보드
- 전체 질문 모니터링 화면
- 실시간 질문 목록 (필터링 전)
- 민감 질문 자동 하이라이팅

#### 4.2 질문 관리 기능
- 질문 숨기기/복구
- 숨김 사유 기록
- 숨긴 질문 목록 관리

#### 4.3 AI 악플 필터링 (Supabase Edge Function)
- 욕설/비속어 사전 기반 필터 (서버)
- 정규식 기반 변형 욕설 탐지
- 위험도 스코어링 시스템
- 필터링 로그 저장

#### 4.4 셀럽 계정 관리
- 셀럽 프로필 수정 권한
- 답변 대리 작성 (선택)
- 구독자 통계 확인

#### 4.5 리포트 및 통계
- 필터링된 질문 통계
- 일일/주간/월간 리포트
- 이슈성 키워드 트렌드

### 완료 조건
- [ ] 매니저가 모든 질문 실시간 모니터링 가능
- [ ] 문제 질문을 숨기면 셀럽 화면에서 즉시 제거
- [ ] Edge Function으로 욕설 자동 탐지율 90% 이상
- [ ] 필터링 로그가 DB에 정상 저장
- [ ] 주간 리포트 PDF 다운로드 기능 작동

### 의존성
- M3 (셀럽 기능) 완료 권장 (하지만 병렬 개발 가능)

## 🚀 Milestone 5: 통합 테스트 및 런칭 준비
**기간**: 1.5주  
**목표**: 전체 시스템 안정화 및 배포 준비

### 주요 산출물
#### 5.1 통합 테스트
- 3-Tier 계정 간 상호작용 시나리오 테스트
- 권한 경계 테스트 (RBAC 검증)
- 성능 테스트 (질문 리스트 1초 이내 로딩)
- 실시간 기능 안정성 테스트

#### 5.2 버그 수정 및 최적화
- 크리티컬 버그 해결
- UI/UX 개선
- 코드 리팩토링
- DB 쿼리 최적화

#### 5.3 보안 점검
- SQL Injection 테스트
- XSS 방지 검증
- RLS 정책 재점검
- API Rate Limiting 설정

#### 5.4 배포 준비
- 프로덕션 환경 설정
- 모니터링 도구 설정 (Sentry, Analytics 등)
- 앱스토어/플레이스토어 등록 준비
- 서비스 약관/개인정보처리방침 작성

#### 5.5 문서화
- API 문서
- 사용자 가이드
- 관리자 매뉴얼
- 장애 대응 프로세스

### 완료 조건
- [ ] 전체 시나리오 테스트 통과 (팬→질문→좋아요→셀럽 답변→매니저 모니터링)
- [ ] 크리티컬 버그 0건
- [ ] 보안 점검 리스트 100% 완료
- [ ] 앱스토어 심사 제출 준비 완료
- [ ] 베타 테스터 10명 이상 테스트 완료

### 의존성
- M1, M2, M3, M4 모두 완료 필수

## 📊 의존성 다이어그램

```
M0 (Foundation)
    ↓
M1 (Auth & Infrastructure)
    ↓
M2 (Fan Features) ← 필수 선행
    ↓
M3 (Celebrity Features) ← M2 데이터 필요
    ↓ (병렬 가능)
M4 (Manager & Security) ← M3 권장, M2 필수
    ↓
M5 (Integration & Launch) ← 전체 완료 필수
```

## ⚠️ 리스크 관리

### High Risk
- **M2-M3 간 데이터 의존성**: 팬 질문이 없으면 셀럽 기능 테스트 불가
  - **완화 전략**: M2 완료 후 시드 데이터 생성
  
- **M4 AI 필터링 정확도**: 90% 탐지율 미달성 가능
  - **완화 전략**: 사전 기반 + 정규식 하이브리드 접근, 지속적 사전 업데이트

### Medium Risk
- **실시간 기능 안정성**: Supabase Realtime 장애
  - **완화 전략**: Polling 폴백 메커니즘 구현

- **멀티 플랫폼 호환성**: iOS/Android/Web 각각 다른 이슈
  - **완화 전략**: 각 플랫폼별 1명씩 QA 담당 배정

## 🔄 Agile 프로세스 적용

### Sprint 구조
- 1 Sprint = 1주
- 각 Milestone 내에서 2-3개 Sprint로 분할
- 매 Sprint 종료 시 데모 진행

### Daily Standup
- 매일 15분
- 어제 한 일 / 오늘 할 일 / 블로커

### Sprint Review & Retrospective
- 매주 금요일
- 완료된 기능 시연
- 회고 및 개선점 도출

## ✅ Definition of Done (DoD)

각 Milestone은 다음 조건을 모두 만족해야 완료로 간주:

1. ✅ 모든 기능 요구사항 구현 완료
2. ✅ 단위 테스트 작성 (커버리지 70% 이상)
3. ✅ 코드 리뷰 승인
4. ✅ 통합 테스트 통과
5. ✅ UI/UX 디자인 가이드 준수
6. ✅ 기술 문서 작성 완료
7. ✅ 스테이징 환경 배포 및 검증
