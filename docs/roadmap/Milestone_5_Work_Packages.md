## WP-5.1: 통합 테스트 및 시나리오 검증

### 목표

전체 시스템의 핵심 사용자 플로우를 검증하고, 3-Tier 계정 간 상호작용이 정상 작동하는지 확인

### 사용자 가치

시스템이 실제 사용 환경에서 안정적으로 작동함을 보장하여, 런칭 후 사용자 불만을 최소화하고 신뢰도를 확보

### 작업 내용

1. **테스트 시나리오 작성**
   - `docs/testing/scenarios/` 디렉토리 생성
   - 핵심 플로우 시나리오 문서화
     - 시나리오 1: 팬 회원가입 → 질문 작성 → 좋아요 → 셀럽 답변 확인
     - 시나리오 2: 셀럽 로그인 → Top 질문 확인 → 답변 작성 → 피드 확인
     - 시나리오 3: 매니저 로그인 → 질문 모니터링 → 욕설 필터링 확인 → 리포트 생성
     - 시나리오 4: 권한 경계 테스트 (팬이 매니저 기능 접근 시도)
     - 시나리오 5: 실시간 업데이트 테스트 (질문 작성 → 실시간 반영)
     - 시나리오 6: Edge Case 테스트 (네트워크 오류, 권한 없음 등)

2. **자동화 테스트 작성**
   - `tests/integration/` 디렉토리 생성
   - Flutter Integration Test 작성
     - `test_integration_fan_flow.dart`: 팬 플로우 테스트
       - 회원가입 → 질문 작성 → 좋아요 → 구독
     - `test_integration_celebrity_flow.dart`: 셀럽 플로우 테스트
       - 로그인 → 질문 확인 → 답변 작성 → 피드 작성
     - `test_integration_manager_flow.dart`: 매니저 플로우 테스트
       - 로그인 → 질문 모니터링 → 질문 숨기기 → 리포트 확인
     - `test_integration_rbac.dart`: 권한 경계 테스트
       - 팬이 매니저 기능 접근 시도 → 차단 확인
       - 셀럽이 팬 전용 기능 접근 → 차단 확인
     - `test_integration_realtime.dart`: 실시간 기능 테스트
       - 질문 작성 → 실시간 반영 확인
       - 답변 작성 → 실시간 반영 확인

3. **수동 테스트 체크리스트**
   - `docs/testing/manual_test_checklist.md` 생성
   - 역할별 테스트 체크리스트 작성
     - 팬 기능 체크리스트
     - 셀럽 기능 체크리스트
     - 매니저 기능 체크리스트
   - 크로스 플랫폼 테스트 (iOS/Android/Web)
   - 에지 케이스 테스트 (네트워크 오류, 권한 없음, 데이터 없음 등)

4. **성능 테스트**
   - 질문 리스트 로딩 시간 측정 (목표: 1초 이내)
   - 대량 데이터 테스트 (1000개 이상 질문)
   - 동시 사용자 테스트 (10명 이상)
   - 메모리 누수 검사 (Flutter DevTools 활용)
   - 네트워크 지연 시뮬레이션 테스트

5. **테스트 결과 문서화**
   - `docs/testing/test_results.md` 생성
   - 테스트 결과 기록 및 버그 리포트 작성
   - 성능 벤치마크 결과 기록
   - 테스트 커버리지 리포트 생성

### 완료 조건

- [ ] 모든 핵심 시나리오 테스트 통과
- [ ] 권한 경계 테스트 통과 (팬/셀럽/매니저 권한 분리 확인)
- [ ] 성능 테스트 통과 (질문 리스트 1초 이내 로딩)
- [ ] 실시간 기능 안정성 확인 (10분 이상 지속 테스트)
- [ ] 크로스 플랫폼 테스트 완료 (iOS/Android/Web)
- [ ] 테스트 결과 문서화 완료
- [ ] 발견된 버그 우선순위 분류 완료

### 산출물

- 통합 테스트 코드
- 테스트 시나리오 문서
- 수동 테스트 체크리스트
- 테스트 결과 리포트
- 성능 벤치마크 리포트
- 버그 리포트 및 우선순위 분류

## WP-5.2: 성능 최적화 및 크리티컬 버그 수정

### 목표

시스템 성능을 최적화하고 크리티컬 버그를 해결하여 안정적이고 빠른 서비스 제공

### 사용자 가치

빠르고 안정적인 서비스 경험 제공으로 사용자 만족도 향상 및 이탈률 감소

### 작업 내용

1. **크리티컬 버그 수정**
   - WP-5.1에서 발견된 크리티컬 버그 우선순위 분류
   - 버그 수정 및 재테스트
   - 버그 수정 이력 문서화 (`docs/bugfixes/critical_bugs.md`)
   - 핫픽스 배포 프로세스 수립

2. **데이터베이스 쿼리 최적화**
   - 느린 쿼리 식별 (Supabase Dashboard 쿼리 분석)
   - 인덱스 추가 (`supabase/migrations/012_add_performance_indexes.sql`)
     - `questions.created_at` 인덱스
     - `questions.user_id` 인덱스
     - `questions.is_hidden` 인덱스
     - `filtering_logs.filtered_at` 인덱스
     - `answers.question_id` 인덱스
   - N+1 쿼리 문제 해결
   - 쿼리 결과 캐싱 전략 수립
   - 불필요한 데이터 조회 최소화

3. **코드 리팩토링**
   - 중복 코드 제거
   - 복잡한 함수 분리
   - 네이밍 일관성 개선
   - 주석 및 문서화 개선
   - 코드 스타일 통일 (dart format, lint)

4. **UI/UX 개선**
   - 로딩 상태 개선 (스켈레톤 UI 등)
   - 에러 메시지 사용자 친화적 개선
   - 애니메이션 최적화 (과도한 애니메이션 제거)
   - 접근성 개선 (스크린 리더 지원 등)
   - 반응형 디자인 개선

5. **메모리 및 성능 최적화**
   - 이미지 최적화 (캐싱, 리사이징)
   - 리스트 렌더링 최적화 (ListView.builder 활용)
   - 불필요한 rebuild 방지 (const 위젯 활용, Provider 최적화)
   - 메모리 누수 검사 및 수정
   - 앱 시작 시간 최적화

6. **성능 모니터링 설정**
   - Flutter DevTools 연동
   - 성능 메트릭 수집 (앱 시작 시간, 화면 전환 시간 등)
   - 성능 대시보드 구축
   - 성능 저하 알림 설정

### 완료 조건

- [ ] 크리티컬 버그 0건
- [ ] 질문 리스트 로딩 시간 1초 이내 달성
- [ ] 데이터베이스 쿼리 최적화 완료 (느린 쿼리 제거)
- [ ] 메모리 누수 없음 확인
- [ ] UI/UX 개선사항 적용 완료
- [ ] 성능 벤치마크 목표 달성
- [ ] 코드 리팩토링 완료 (코드 품질 향상)

### 산출물

- 최적화된 코드베이스
- 데이터베이스 인덱스 마이그레이션

- 버그 수정 이력 문서

## WP-5.3: 보안 점검 및 강화

### 목표

보안 취약점을 점검하고 강화하여 안전한 서비스 제공 및 사용자 데이터 보호

### 사용자 가치

사용자 데이터 보호 및 서비스 신뢰도 향상으로 장기적인 사용자 확보

### 작업 내용

1. **SQL Injection 방지 검증**
   - Supabase 클라이언트 사용 시 파라미터화된 쿼리 확인
   - 직접 SQL 쿼리 사용 금지 확인
   - 테스트 쿼리 작성 및 실행
   - SQL Injection 시도 테스트

2. **XSS (Cross-Site Scripting) 방지 검증**
   - 사용자 입력 데이터 이스케이프 처리 확인
   - Flutter의 기본 위젯 사용 시 자동 이스케이프 확인
   - HTML 렌더링 시 sanitization 확인 (있는 경우)
   - XSS 공격 시도 테스트

3. **RLS (Row Level Security) 정책 재점검**
   - `supabase/migrations/013_verify_rls_policies.sql` 생성
   - 모든 테이블의 RLS 정책 검증
     - `users` 테이블 RLS 검증
     - `questions` 테이블 RLS 검증
     - `answers` 테이블 RLS 검증
     - `filtering_logs` 테이블 RLS 검증
   - 역할별 접근 권한 테스트
   - 권한 우회 시도 테스트

4. **API Rate Limiting 설정**
   - Supabase Edge Function Rate Limiting 설정
   - 클라이언트 측 요청 제한 로직 구현
   - DDoS 공격 방지 전략 수립
   - Rate Limiting 테스트

5. **인증 및 세션 관리 강화**
   - JWT 토큰 만료 시간 검증
   - 세션 타임아웃 설정 확인
   - 로그아웃 시 토큰 무효화 확인
   - 토큰 갱신 프로세스 검증

6. **데이터 암호화 확인**
   - 민감 정보 암호화 확인 (비밀번호, 개인정보 등)
   - HTTPS 통신 확인
   - 데이터베이스 암호화 확인 (Supabase 기본 암호화 활용)
   - 전송 중 데이터 암호화 확인

7. **보안 점검 체크리스트 작성**
   - `docs/security/security_checklist.md` 생성
   - OWASP Top 10 기반 체크리스트 작성
   - 보안 점검 결과 문서화
   - 보안 개선 사항 우선순위 분류

### 완료 조건

- [ ] SQL Injection 테스트 통과
- [ ] XSS 방지 검증 완료
- [ ] RLS 정책 100% 검증 완료
- [ ] API Rate Limiting 설정 완료
- [ ] 인증 및 세션 관리 강화 완료
- [ ] 데이터 암호화 확인 완료
- [ ] 보안 점검 체크리스트 100% 완료
- [ ] 보안 점검 결과 문서화 완료

### 산출물

- 보안 강화 코드
- RLS 정책 검증 마이그레이션
- 보안 점검 체크리스트
- 보안 점검 결과 리포트
- 보안 개선 사항 문서

## WP-5.4: 프로덕션 배포 환경 구축

### 목표

프로덕션 환경을 구축하고 모니터링 도구를 설정하여 안정적인 서비스 운영 준비

### 사용자 가치

안정적이고 모니터링 가능한 서비스 제공으로 장애 대응 시간 단축 및 서비스 가용성 향상

### 작업 내용

1. **프로덕션 환경 설정**
   - Supabase 프로덕션 프로젝트 설정
   - 환경 변수 설정 (`.env.production`)
   - 프로덕션 데이터베이스 마이그레이션 실행
   - Edge Function 프로덕션 배포
   - 프로덕션 환경 검증

2. **모니터링 도구 설정**
   - Sentry 연동 (`lib/core/monitoring/sentry_config.dart`)
     - 에러 추적 설정
     - 성능 모니터링 설정
     - 릴리스 추적 설정
     - 사용자 컨텍스트 설정
   - Analytics 설정 (선택: Firebase Analytics, Mixpanel 등)
   - 로깅 시스템 구축 (`lib/core/logging/logger.dart`)
     - 로그 레벨 설정 (DEBUG, INFO, WARNING, ERROR)
     - 로그 포맷 통일
     - 프로덕션 로그 필터링

3. **앱스토어/플레이스토어 등록 준비**
   - 앱 아이콘 및 스플래시 스크린 준비
   - 앱 스토어 스크린샷 준비 (iOS/Android)
   - 앱 설명 및 키워드 작성
   - 개인정보처리방침 작성 (`docs/legal/privacy_policy.md`)
   - 서비스 약관 작성 (`docs/legal/terms_of_service.md`)
   - 연령 등급 설정
   - 앱스토어 심사 가이드라인 준수 확인

4. **CI/CD 파이프라인 구축**
   - GitHub Actions 또는 GitLab CI 설정
   - 자동 테스트 실행
   - 자동 빌드 및 배포
   - 환경별 배포 전략 수립 (개발/스테이징/프로덕션)
   - 배포 롤백 프로세스 수립

5. **백업 및 복구 전략**
   - 데이터베이스 백업 설정 (Supabase 자동 백업 활용)
   - 백업 복구 프로세스 문서화 (`docs/operations/backup_recovery.md`)
   - 재해 복구 계획 수립 (`docs/operations/disaster_recovery.md`)
   - 백업 테스트 실행

6. **성능 모니터링 대시보드**
   - Supabase Dashboard 모니터링 설정
   - 주요 메트릭 추적 (API 호출 수, 에러율, 응답 시간 등)
   - 알림 설정 (에러 발생 시 알림)
   - 성능 대시보드 구축

- **앱스토어 심사 지연**: 심사 거절 또는 수정 요청

  - **완화 전략**: 문서화 체크리스트 작성, 단계적 문서화, 리뷰 프로세스 수립

  - **완화 전략**: 조기 베타 테스터 모집, 인센티브 제공, 내부 테스트 강화
