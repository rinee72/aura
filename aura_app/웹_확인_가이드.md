# Flutter 웹 앱 Chrome에서 확인하기

## 🚀 현재 상태

Flutter 웹 앱이 Chrome에서 실행 중입니다!

## 📍 확인 방법

### 방법 1: 자동으로 열린 Chrome 창 확인
- Flutter가 자동으로 Chrome을 열었을 수 있습니다
- 작업 표시줄에서 Chrome 아이콘을 확인하세요
- 새로 열린 Chrome 탭에서 Flutter 앱이 표시되어야 합니다

### 방법 2: 직접 URL 접속
Flutter 웹 앱은 보통 다음 URL 중 하나에서 실행됩니다:

1. **http://localhost:50000** (또는 다른 포트)
2. **http://127.0.0.1:50000**

**확인 방법:**
- Chrome 주소창에 `localhost` 입력
- 또는 `http://localhost:50000` 입력 (포트 번호는 다를 수 있음)

### 방법 3: 컴포넌트 카탈로그 확인
앱이 열리면:

1. **홈 화면**에서 AppBar 오른쪽의 **팔레트 아이콘(🎨)** 클릭
   - 개발 환경에서만 표시됩니다
   - 또는 주소창에 `/showcase` 추가: `http://localhost:50000/showcase`

2. **컴포넌트 카탈로그 페이지**에서 확인할 수 있는 내용:
   - ✅ **Colors**: Primary, Secondary, Error, Success 등
   - ✅ **Typography**: H1~H6, Body1~3, Label, Caption 등
   - ✅ **Spacing**: XS, SM, MD, LG, XL, XXL
   - ✅ **Buttons**: Primary, Secondary, Outlined, Text, Loading, Disabled
   - ✅ **Text Fields**: 기본, 아이콘 포함, 에러 상태, 비활성화
   - ✅ **Cards**: 기본, 제목 포함, 클릭 가능
   - ✅ **Loading**: 다양한 크기와 메시지
   - ✅ **Error**: 에러 메시지 표시

## 🔍 확인할 내용

### WP-0.5 디자인 시스템 확인 항목:

1. **색상 시스템** (Colors)
   - Primary, Secondary 색상이 올바르게 표시되는지
   - Error, Success, Warning, Info 색상 확인

2. **타이포그래피** (Typography)
   - 각 헤딩 레벨(H1~H6)의 크기와 스타일
   - Body 텍스트 스타일
   - Label, Caption, Button 텍스트

3. **간격 시스템** (Spacing)
   - XS, SM, MD, LG, XL, XXL 간격이 시각적으로 구분되는지

4. **컴포넌트**
   - 버튼의 다양한 variant와 상태
   - 텍스트 필드의 다양한 상태
   - 카드 컴포넌트
   - 로딩 인디케이터
   - 에러 메시지

## 🛠️ 문제 해결

### Chrome이 열리지 않았다면:

1. **수동으로 Chrome 열기**:
   ```
   Chrome 주소창에 입력: http://localhost:50000
   ```

2. **포트 번호 확인**:
   - Flutter가 사용하는 포트는 터미널 로그에서 확인 가능
   - 보통 50000번대 포트 사용

3. **다시 실행**:
   ```powershell
   cd c:\modu\aura_app
   flutter run -d chrome
   ```

## 📝 참고

- 개발 환경에서만 AppBar에 팔레트 아이콘이 표시됩니다
- 컴포넌트 카탈로그는 `/showcase` 라우트로 접근 가능합니다
- Hot Reload가 활성화되어 있어 코드 변경 시 자동으로 반영됩니다
