# Flutter 앱 실행 문제 해결 가이드

## 확인된 상황

✅ **Git은 정상적으로 설치되어 있습니다**
- 경로: `C:\Program Files\Git\cmd\git.exe`
- PATH에도 포함되어 있음

❌ **하지만 Flutter가 Git을 찾지 못하는 오류 발생**
- `Error: Unable to find git in your PATH.`
- Flutter Daemon이 시작되지 않음

## 원인 분석

Cursor/VS Code가 시작될 때 환경 변수를 캐시하는데, Git이 나중에 설치되었거나 PATH가 변경된 경우 Cursor가 이를 인식하지 못할 수 있습니다.

## 해결 방법 (순서대로 시도)

### 방법 1: Cursor 완전 재시작 (가장 효과적)

1. **Cursor 완전히 종료**
   - 모든 Cursor 창 닫기
   - 작업 관리자에서 Cursor 프로세스가 모두 종료되었는지 확인

2. **새 PowerShell 창에서 Git 확인**
   ```powershell
   git --version
   ```
   - 버전이 표시되면 Git이 정상 작동 중

3. **Cursor 다시 실행**
   - 프로젝트 폴더 열기
   - Command Palette (Ctrl+Shift+P) → "Dart: Pub Get" 실행
   - Command Palette → "Dart: Restart Analysis Server" 실행

### 방법 2: Flutter 캐시 정리

새 PowerShell 창에서 실행:
```powershell
cd c:\modu\aura_app
C:\flutter\bin\flutter.bat clean
C:\flutter\bin\flutter.bat pub get
```

### 방법 3: 환경 변수 강제 새로고침

1. Windows 검색에서 "환경 변수" 검색
2. "시스템 환경 변수 편집" 선택
3. "환경 변수" 버튼 클릭
4. "시스템 변수"에서 "Path" 확인
5. `C:\Program Files\Git\cmd`가 있는지 확인
6. 없으면 추가, 있으면 그대로 두기
7. **모든 창 닫기**
8. **Cursor 완전히 재시작**

### 방법 4: Flutter가 Git 없이도 작동하도록 설정

현재 `pubspec.yaml`에는 Git 의존성이 없으므로, Flutter가 Git을 찾지 못해도 작동할 수 있습니다. 하지만 Flutter 자체가 Git을 찾으려고 시도하므로 완전한 해결책은 아닙니다.

## 빠른 해결 체크리스트

1. ✅ Git 설치 확인 (완료 - `C:\Program Files\Git\cmd\git.exe` 존재)
2. ✅ PATH 확인 (완료 - PATH에 Git 포함)
3. ⏳ **Cursor 완전 재시작** ← 지금 해야 할 일!
4. ⏳ Command Palette → "Dart: Pub Get" 실행
5. ⏳ Command Palette → "Dart: Restart Analysis Server" 실행
6. ⏳ 앱 실행: `flutter run -d chrome --dart-define=ENVIRONMENT=development`

## 참고

- PowerShell 인코딩 오류는 환경 문제이며 Flutter 코드 오류가 아닙니다
- Git이 PATH에 있으므로 Cursor 재시작 후 정상 작동할 가능성이 높습니다
- Flutter Daemon은 Dart/Flutter 확장이 자동으로 시작하는 백그라운드 프로세스입니다
