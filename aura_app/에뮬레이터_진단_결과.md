### ✅ 1. 하이퍼바이저 활성화 여부

**테스트 결과**: ⚠️ **확인 불가 (관리자 권한 필요)**

**시도한 명령:**
- `Get-WindowsOptionalFeature` - 관리자 권한 필요로 실패
- `dism.exe /online /get-featureinfo` - 관리자 권한 필요로 실패
- `bcdedit /enum` - 결과 없음

**상태:**
- ❓ **Windows Hypervisor Platform**: 확인 불가 (수동 확인 필요)
- ❓ **Hyper-V**: 확인 불가 (수동 확인 필요)
- ❓ **BIOS 가상화 설정**: 확인 불가 (BIOS 접근 필요)

**권장 조치:**
1. **Windows 기능 확인**:
   - `Win + R` → `appwiz.cpl` 입력
   - "Windows 기능 켜기/끄기" 클릭
   - "Windows Hypervisor Platform" 또는 "Hyper-V" 체크 여부 확인

2. **BIOS 확인**:
   - 컴퓨터 재시작 시 BIOS 진입 (보통 F2, F10, Del 키)
   - Virtualization Technology (VT-x) 또는 AMD-V 활성화 여부 확인

## 🔧 우선순위별 해결 방법

### 🔴 우선순위 1: 하이퍼바이저 확인 및 활성화

**확인 방법:**
1. **Windows 기능 확인**:
   ```
   Win + R → appwiz.cpl → "Windows 기능 켜기/끄기"
   → "Windows Hypervisor Platform" 체크 여부 확인
   ```

2. **활성화 방법** (비활성화되어 있다면):
   - "Windows 기능 켜기/끄기"에서 "Windows Hypervisor Platform" 체크
   - 재부팅

**또는 Intel HAXM 설치:**
- Android Studio > SDK Manager > SDK Tools
- "Intel x86 Emulator Accelerator (HAXM installer)" 설치

---

### 🟡 우선순위 2: 한글 경로 문제 해결

**해결 방법:**

1. **Android SDK 경로 변경** (권장):
   - Android Studio > Settings > Appearance & Behavior > System Settings > Android SDK
   - Android SDK Location을 한글이 없는 경로로 변경 (예: `C:\Android\Sdk`)

2. **AVD 경로 변경**:
   - 환경 변수 `ANDROID_AVD_HOME` 설정
   - 한글이 없는 경로로 설정 (예: `C:\Android\avd`)

3. **임시 해결책**:
   - Android Studio에서 직접 에뮬레이터 실행 (경로 문제 우회 가능)

---

### 🟢 우선순위 3: Android Studio에서 직접 실행

**가장 빠른 해결책:**
1. Android Studio 열기
2. Tools > Device Manager
3. Pixel 5 에뮬레이터 재생 버튼 클릭
4. 에러 메시지 확인 (있다면)

**이유:**
- Android Studio는 경로 문제를 더 잘 처리함
- 더 자세한 에러 메시지 제공
- 하이퍼바이저 문제도 명확히 표시
