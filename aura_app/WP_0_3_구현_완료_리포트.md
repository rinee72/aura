### 보조 브랜치
- **`feature/*`**: 기능 개발 브랜치

- **`release/*`**: 릴리스 준비 브랜치

### 워크플로우
1. `develop`에서 `feature/*` 브랜치 생성
2. 기능 개발 완료 후 `develop`에 PR
3. 코드 리뷰 및 CI 통과 후 머지
4. 릴리스 준비 시 `release/*` 브랜치 생성
5. 릴리스 완료 후 `main`과 `develop`에 머지

## 📝 커밋 메시지 규칙

### 형식
```
<type>(<scope>): <subject>

<body>

<footer>
```

### Type
- `feat`: 새 기능 추가
- `fix`: 버그 수정
- `docs`: 문서 수정
- `style`: 코드 포맷팅
- `refactor`: 리팩토링
- `test`: 테스트 코드
- `chore`: 빌드 설정, 패키지 관리

### 예시
```bash
feat(auth): 사용자 로그인 기능 추가
fix(questions): 질문 목록 무한 스크롤 버그 수정
docs(readme): README 업데이트
```
